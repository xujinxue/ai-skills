# Paper Types

Every note keeps the same base skeleton, but emphasis changes by paper type.

## `AI_method`

Emphasize:
- problem setup
- method architecture
- training recipe
- comparison baselines
- ablations
- tradeoffs and reuse potential

Common subheadings:
- `方法直觉`
- `模型结构`
- `训练与推理`
- `实验设计`

## `benchmark_or_dataset`

Emphasize:
- task decomposition
- annotation pipeline
- coverage and bias
- evaluation scope
- what the benchmark actually measures

Common subheadings:
- `任务拆分`
- `数据构建`
- `评测维度`
- `benchmark 真正测到了什么`

## `clinical_or_psychology_empirical`

Emphasize:
- sample source
- variables or scales
- task definition
- analysis pipeline
- clinical meaning
- generalization boundary

Common subheadings:
- `样本与招募`
- `变量与量表`
- `任务定义`
- `临床意义`

## `humanities_or_social_science`

Emphasize:
- object of study
- materials and corpus
- theoretical framing
- argument structure
- conceptual contribution
- interpretive limitations

Common subheadings:
- `理论框架`
- `材料来源`
- `论证路径`
- `概念贡献`

## Selection Rule

Choose one primary type, then adapt details within the note.
Do not remove the base note skeleton unless a section truly does not apply.
